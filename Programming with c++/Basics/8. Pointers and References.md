#### 1. Pointer Basics (\*, &)

- **Pointer** is a variable that stores the memory address of another variable.
- **Operators:**
    - `&` — _address-of operator_ (gets memory address).
    - `*` — _dereference operator_ (accesses the value at that address).

- **Example:**
```cpp
int x = 10;
int* p = &x;       // p stores address of x
cout << *p;        // outputs 10 (value at address)
```

#### 2. Null Pointers
- A pointer that **does not point to any valid memory location**.
- In modern C++, use nullptr instead of NULL or 0.

- **Example:**
```cpp
int* ptr = nullptr;   // safe initialization
if (ptr == nullptr) 
{
    cout << "Pointer is empty\n";
}
```

#### 3. Pointer Arithmetic
- You can move a pointer to access adjacent memory locations.
- Arithmetic works in steps of the **type size** (not bytes directly).

- **Example:**
```cpp
int arr[3] = {10,20,30};
int* p = arr;
cout << *(p+1);   // outputs 20
p++;              // now points to arr[1]  
```

- **Important:** Avoid arithmetic on pointers that are not part of an array or allocated block.

#### 4. Pointers and Arrays
- **Array name behaves like a pointer** to its first element.
- arr[i] is equivalent to *(arr + i).

- **Example:**
```cpp
int arr[3] = {1,2,3};
int* p = arr;
cout << *(p+2);   // outputs 3
```

#### 5. References vs Pointers

- **Reference (&):**
    - Must be initialized at declaration.
    - Cannot be reseated to another object.
    - Acts as an alias to an existing variable.
    
- **Pointer:**
    - Can be uninitialized or nullptr.
    - Can be reassigned to point elsewhere.

- **Example:**
```cpp
int x = 10, y = 20;
int& ref = x;   // reference to x
int* ptr = &x;  // pointer to x
ref = y;        // changes value of x to 20
ptr = &y;       // pointer now points to y
```

#### 6. Dynamic Memory Allocation (new and delete)

- **Allocate memory at runtime:**
```cpp
int* p = new int(5);    // allocate single int
int* arr = new int[10]; // allocate array of 10 ints
```

- **Free memory explicitly to avoid leaks:**
```cpp
delete p;        // free single int
delete[] arr;    // free array
```

#### 7. Dangling Pointers and Memory Leaks

- **Dangling Pointer:** Pointer still points to freed or invalid memory.
```cpp
int* p = new int(10);
delete p;
// p is dangling now — accessing *p is undefined behavior!
```

- **Memory Leak:** Forgetting to delete dynamically allocated memory.
- **Best Practice:**
    - Always set pointer to nullptr after delete.
    - Use **smart pointers** (std::unique_ptr, std::shared_ptr) in modern C++.
